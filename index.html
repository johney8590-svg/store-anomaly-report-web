門市清單維護

每行一個門市名稱（可直接新增、修改、刪除或整批貼上匯入）。按「儲存門市」後存到雲端，全門市即時同步。

品名主檔維護

每行一筆，格式「品名,料號」（用半形逗號分隔；料號可留空）。可整批貼上匯入。按「儲存品名」後全門市下拉同步。

LINE 通知設定

採用 LINE 官方帳號 Messaging API 推播。填好下面兩格、按「儲存設定」後即可使用；可先用「發送測試訊息」確認。

送出案件時，系統會把這裡的設定交給雲端後端代發 LINE（瀏覽器無法直接呼叫 LINE）。設定值會存在本機瀏覽器，請勿在公用電腦留存。
可用代入欄位：{caseNo} 案件編號 · {name} 品名 · {qty} 異常數量 · {mat} 異常料號 · {mfg} 製造日期 · {exp} 有效日期 · {date} 異常日期 · {desc} 異常說明 · {cls} 異常分類 · {store} 門市 · {link} 連結。製造／有效日期沒填會自動留空。

串接步驟

1建立 LINE 官方帳號：到 developers.line.biz 建立 Provider 與 Messaging API channel，取得長期 Channel access token。
2取得推播目標 ID：把官方帳號加入要通知的群組／好友，透過 Webhook 事件取得 groupId 或 userId。
3填入上方兩格並儲存：把長期 token 與目標 ID 填到上面，按「儲存設定」。送出案件時會由雲端後端（Code.gs）代發，瀏覽器不會直接接觸 LINE。
4訊息內容：系統已自動組好含案號、門市、品名、數量、狀況的文字，並附上照片／影片連結。
